ChangeLog v2.1 

Se agregan paquete con clases para test de GUI.
Se testean funciones basicas, sobretodo aquellas que son input del modelo para asegurarse
que se cumplen las precondiciones de los metodos.
Ver planilla de tracking de testeo anexa.

Clase Principal:
- A fin de testear los ingresos por inputDialog (JOptionPane) se creo un metodo que 
copia al metodo getFecha original con la salvedad de que reemplaza el uso del inputDialog
por un str que se envia como parametro y simula ser el resultado de dicho elemento.
- Se procede con la misma logica para el metodo que obtiene el motivo de cancelacion
de un pedido.

Clase VentanaMaquinas:
- Con el objetivo de testear el metodo getCantidad se crea metodo sustituto que cambia
el inputDialog por un string pasado como parametro.
